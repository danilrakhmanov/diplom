# Описание дипломного проекта

## Тема

Разработка информационной системы магазина спортивных товаров.

## Цель проекта

Целью является разработка программной системы, которая автоматизирует основные процессы магазина спортивных товаров: учет ассортимента, хранение данных о клиентах, оформление заказов, расчет суммы продажи и контроль складских остатков.

## Задачи

1. Проанализировать предметную область магазина спортивных товаров.
2. Спроектировать структуру базы данных.
3. Реализовать серверную часть с API для работы с товарами, клиентами и заказами.
4. Реализовать пользовательский веб-интерфейс.
5. Проверить корректность оформления заказов и обновления складских остатков.

## Пользователи системы

Основным пользователем является сотрудник магазина. Он работает с каталогом, добавляет клиентов, оформляет продажи и отслеживает товары, которые нужно заказать у поставщика.

## Основные бизнес-процессы

### Учет товаров

Сотрудник просматривает каталог, фильтрует товары по категории, ищет товар по названию или бренду и добавляет новые позиции. Для каждого товара хранится название, категория, бренд, цена, количество на складе и описание.

### Работа с клиентами

В системе хранится клиентская база. Для клиента указываются ФИО, телефон, email и бонусные баллы. После оформления заказа клиенту начисляются бонусы.

### Оформление продажи

Сотрудник выбирает клиента и добавляет товары в заказ. Система проверяет наличие товара на складе, рассчитывает итоговую сумму, сохраняет заказ и уменьшает складские остатки.

### Аналитика

На главной странице отображаются количество товаров, клиентов, заказов, общая выручка, популярные товары и позиции с низким остатком.

## Архитектура

Проект реализован как веб-приложение:

- серверная часть: Python, `http.server`, SQLite;
- клиентская часть: HTML, CSS, JavaScript;
- база данных: SQLite.

Такой подход подходит для учебного проекта, потому что приложение не требует установки внешних библиотек и может быть запущено на локальном компьютере.

## Модель данных

Таблица `categories` хранит категории товаров.

Таблица `products` хранит товары и связана с `categories` по внешнему ключу.

Таблица `customers` хранит покупателей.

Таблица `orders` хранит заказы и связана с `customers`.

Таблица `order_items` хранит состав заказа и связана с `orders` и `products`.

## Перспективы развития

В дальнейшем систему можно расширить авторизацией сотрудников, ролями пользователей, модулем поставщиков, печатью чеков, импортом товаров из Excel и отчетами за период.
